# Core

Это пакет с общими функциями и классами. Так как большая часть кода повторяется, решил вынести эти функции в отдельный пакет, чтобы не дублировать код в каждом приложении. Если вы не используете docker-compose, необходимо вручную установить пакет.

## v0.1.4b

#### RedisClient

- добавлена функция `clear` для очистки всех ключей в Redis связанных с приложением или указанным префиксом
- Добавлено логирование.

#### SQLRepository

- Добавлен параметр `loader_options` для загрузки связанных моделей
- Добавлен параметр `select_columns` для выборки определенных колонок

#### RequestsMakers

- Добавлено логирование.

#### Debug

- Новый пакет с добавлением `LoggerManager` для контроля над логированием с возможностью отправки логов в `NCT-Dispatcher`

## Будущие изменения в v0.2.0

В этой версии будут серьезные изменения в пакетах:

#### sql_repository

- Удален метод delete
- все методы связанные с `load_relations` (загрузкой связанных моделей) будут заменены на `loader_options`
- Удалены `relationships` при инициализации модели

## Подмодули

### dot_env

Пакет для использования переменных .env. Процесс схож с pydantic-settings, но пакет легче, а функционально поддерживаются разные списки.

### fast_decorators

Пакет с декораторами для FastAPI. Сейчас реализованы следующие декораторы:

- `rate_limiter`: ограничивает количество запросов в определенное время
- `cache`: кэширует результаты запросов # работает в тестовом режиме

### fast_depends

Пакет с зависимостями для FastAPI. Сейчас реализованы следующие зависимости:

- `PaginationParams`: параметры для пагинации

### fast_middlewares

Пакет с middleware для fastapi.

- `blocker_check`: проверяет не находится ли пользователь в бане

### fast_routers

Пакет с роутерами для FastAPI. Сейчас реализованы следующие роутеры:

- `utils_router_v1`: роутер с статусом приложения

### redis_client

Пакет реализует работу с Redis, типизируя кэшированные данные.

- `RedisClient`: клиент для работы с Redis
- `RedisDep`: зависимость для работы с fastapi

### requests_makers

Урезанная версия моей библиотеки [RequestsMakers](https://github.com/MaxBro12/Requests-Makers).
Реализован только асинхронный клиент.

### simplejwt

Пакет реализующий мою версию jwt токенов их валидацию и генерацию

- `SimpleJWT`: класс для работы с jwt токенами

### sql_repository

Новая версия [SqlAlchemyExtRepo](https://github.com/MaxBro12/SqlAlchemyExtRepo) вскоре функционал будет добавлен и в библиотеку.

[Подробное README](https://github.com/MaxBro12/src/core/sql_repository/README.md)

- `Repository`: класс для работы с моделями sqlalchemy в формате текстовых запросов - устаревший формат, дальнеёший поддержки не будет
- `RepositoryObj`: класс для работы с моделями sqlalchemy в формате orm
- `DataBaseRepo`: класс для работы со всей базой данных

#### Изменения в 0.2.0

В этой версии будут существенные изменения в функционале объекта `RepositoryObj`. 

###### Удаление метода `delete`

Сейчас метод `delete` работает но вызывает предупреждение.

- `RepositoryObj.delete`: метод будет удален
- `RepositoryObj._delete`: новый метод удаления записи используя фильтр
- `RepositoryObj._delete_obj`: новый метод удаления записи используя объект (ранее `RepositoryObj.delete`)

###### Изменения в методах `add` и `add_many`

Методы `add` и `add_many` были переименованы в `_add` и `_add_many` во избежание конфликтов с методами. Так же были изменены возвращаемые значения методов:

- `RepositoryObj._add`: метод возвращает добавленную модель
- `RepositoryObj._add_many`: метод возвращает список добавленных моделей

### pydantic_misc_models

Пакет с дополнительными моделями pydantic часто встречающихся в сервисах

### spec_time

Пакет с методами для работы с временем.

### trash

Пакет с функциями генерации уникальных строк

## Тестирование

Для тестирования не забудьте установить `requirements-dev.txt` (или `uv sync --extra tests`) и запустить тесты с помощью `pytest .`

Версии пакета включают префиксы:
- `a` - функционал в разработке, некоторые новые методы могут не работать корректно
- `b` - функционал в тестировании, написаны тесты, которые можно запустить через `pytest .`. Функционал уже используется в реальных проектах под присмотром.
